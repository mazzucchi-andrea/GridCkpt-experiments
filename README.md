# Reproducibility PADS 2026

This documents describes how to reproduce the results discussed in the paper:

"Grid Checkpointing in Speculative Simulation"

Accepted to ACM SIGSIM PADS 2026

## Authors & Contacts

* Andrea Mazzucchi <andrea.mazzucchi@tutamail.com>
* Francesco Quaglia <francesco.quaglia@uniroma2.it>

## Requirements

* x86_64 CPU
* 16GB of RAM
* Linux with Kernel Version >= 4.17

The hardware/software configuration used by the authors is:

* CPU: 2 * Intel Xeon Silver 4210
* RAM: 32 GB (16 GB for each NUMA node)
* OS: Ubuntu 25.04 (Kernel Version 6.14)

## Dependencies

For running experiments, processing data and generating figures: 
* ```bash```        (https://www.gnu.org/software/bash/)

Run ```./dep.sh``` to check and install all the dependecies.

## Kick the tires instructions

1. Clone the repository: ```git clone https://github.com/mazzucchi-andrea/GridCkpt-experiments```
2. Move to the artifact folder: ```cd GridCkpt-experiments```
3. Run small tests (~8m): ```./exp.sh kick```

## Article claims

The article has three major claims:

* C1: the overhead of the instrumentation is near-zero and constant w.r.t. write ratio and state size
* C2: Grid Checkpointing outperfoms literature cornerstone solutions with both benchmarks (PHOLD) and real-world case studies (PCS)

## Reproducing the results

The paper has 4 Figures that can be reproduced.
These can be generated by running 3 experiments.
The mapping between claims, experiments, figures and tables are resumed in the following table.

| Claim | Figures            | Experiment           |
|-------|--------------------|----------------------|
| C1    | 4, 5               | instr_cost           |
| C2    | 6                  | phold                |
| C2    | 7                  | pcs                  |

To run an experiment <exp> and process its results, type the following:

 ```nohup ./exp.sh run_<exp> &```

To run all experiments and process their results at once, type the following:

```nohup ./exp.sh &```

The expected runtime of each experiment is detailed in the following table:

| Experiment   | Runtime |
|--------------|---------|
| instr_cost   | ~3m     |
| phold        | ~5h 15m |
| pcs          | ~5h 15m |
| **Total**    | **~10h 30m** |

Once all experiments have been run, you can find each figure at:

| Figure      | Path              |
|-------------|-------------------|
| 4           | figures/fig4.png  |
| 5           | figures/fig5.png  |
| 6           | figures/fig6.png  |
| 7           | figures/fig7.png  |
